═══════════════════════════════════════════════════════════════════
  CONFIGURACIÓN DE FUNCIONALIDADES AI - PASO A PASO
═══════════════════════════════════════════════════════════════════

┌─────────────────────────────────────────────────────────────────┐
│ PASO 1: OBTENER API KEY DE OPENAI (5 minutos)                  │
└─────────────────────────────────────────────────────────────────┘

1. Abre tu navegador y ve a:
   https://platform.openai.com/api-keys

2. Si no tienes cuenta:
   - Haz clic en "Sign up"
   - Crea una cuenta con tu email
   - Verifica tu email

3. Si ya tienes cuenta:
   - Haz clic en "Log in"
   - Ingresa tus credenciales

4. Una vez dentro:
   - Haz clic en "Create new secret key"
   - Dale un nombre (ej: "workflow-system")
   - Copia la key completa (empieza con sk-)
   - ⚠️ IMPORTANTE: Guárdala, no la podrás ver de nuevo

5. Verifica que tengas créditos:
   - Ve a "Usage" en el menú
   - Deberías tener $5 gratis si es cuenta nueva


┌─────────────────────────────────────────────────────────────────┐
│ PASO 2: CONFIGURAR LA API KEY EN EL PROYECTO (2 minutos)       │
└─────────────────────────────────────────────────────────────────┘

1. Abre tu editor de código (VS Code, etc.)

2. Navega a:
   diagram/src/environments/environment.ts

3. Busca esta línea:
   openaiApiKey: 'TU_API_KEY_AQUI'

4. Reemplaza 'TU_API_KEY_AQUI' con tu API key real:
   openaiApiKey: 'sk-proj-abc123xyz...'

5. Guarda el archivo (Ctrl+S o Cmd+S)

6. Repite lo mismo en:
   diagram/src/environments/environment.prod.ts

7. Verifica que quedó así:
   ✅ export const environment = {
        production: false,
        apiUrl: 'http://localhost:8080/api',
        openaiApiKey: 'sk-proj-tu-key-real-aqui'
      };


┌─────────────────────────────────────────────────────────────────┐
│ PASO 3: INSTALAR DEPENDENCIAS (1 minuto)                       │
└─────────────────────────────────────────────────────────────────┘

1. Abre una terminal/consola

2. Navega a la carpeta del frontend:
   cd diagram

3. Instala las dependencias:
   npm install

4. Espera a que termine (puede tomar 1-2 minutos)


┌─────────────────────────────────────────────────────────────────┐
│ PASO 4: INICIAR EL BACKEND (1 minuto)                          │
└─────────────────────────────────────────────────────────────────┘

1. Abre una NUEVA terminal/consola

2. Navega a la carpeta del backend:
   cd backend/backend

3. Inicia el backend:
   mvn spring-boot:run

4. Espera a ver:
   "Started BackendApplication in X seconds"

5. El backend estará en:
   http://localhost:8080


┌─────────────────────────────────────────────────────────────────┐
│ PASO 5: INICIAR EL FRONTEND (1 minuto)                         │
└─────────────────────────────────────────────────────────────────┘

1. En la terminal del frontend (donde hiciste npm install):
   npm start

2. Espera a ver:
   "✔ Browser application bundle generation complete"

3. El navegador debería abrirse automáticamente en:
   http://localhost:4200

4. Si no se abre, ábrelo manualmente


┌─────────────────────────────────────────────────────────────────┐
│ PASO 6: PROBAR LAS FUNCIONALIDADES (5 minutos)                 │
└─────────────────────────────────────────────────────────────────┘

═══════════════════════════════════════════════════════════════════
FUNCIONALIDAD 1: CHATBOT AI
═══════════════════════════════════════════════════════════════════

1. Inicia sesión en el sistema:
   - Usuario: admin@example.com
   - Contraseña: (tu contraseña)

2. Busca el widget MORADO en la esquina inferior derecha
   (tiene un ícono de robot)

3. Haz clic en el widget

4. Escribe una pregunta:
   "¿Cómo creo una nueva política?"

5. Presiona Enter o haz clic en el botón de enviar

6. ✅ ÉXITO: El chatbot responde con información útil

7. Prueba otras preguntas:
   - "¿Qué son los cuellos de botella?"
   - "¿Cómo funciona el editor de diagramas?"


═══════════════════════════════════════════════════════════════════
FUNCIONALIDAD 2: BIBLIOTECA DE PLANTILLAS AI
═══════════════════════════════════════════════════════════════════

1. En el menú lateral izquierdo, busca:
   "Plantillas AI" (ícono de estrella)

2. Haz clic en "Plantillas AI"

3. Verás 5 plantillas predefinidas

4. En la barra de búsqueda, escribe:
   "necesito un flujo para instalación eléctrica"

5. Haz clic en el botón "Buscar con AI"

6. ✅ ÉXITO: El sistema encuentra plantillas relevantes

7. Haz clic en "Vista Previa" en una plantilla

8. Verás el flujo completo

9. Haz clic en "Usar Plantilla"

10. ✅ ÉXITO: Se crea una nueva política desde la plantilla


═══════════════════════════════════════════════════════════════════
FUNCIONALIDAD 3: ASISTENTE DE VOZ
═══════════════════════════════════════════════════════════════════

1. Ve a "Políticas" en el menú

2. Crea una nueva política o edita una existente

3. Haz clic en "Diagrama" para abrir el editor

4. Busca el botón MORADO con micrófono
   (esquina inferior derecha)

5. Haz clic en el botón de micrófono

6. El navegador pedirá permiso para usar el micrófono
   → Haz clic en "Permitir"

7. Di claramente:
   "Agregar nodo validación"

8. ✅ ÉXITO: Se crea un nodo llamado "validación"

9. Prueba otros comandos:
   - "Agregar calle cliente"
   - "Guardar diagrama"
   - "Ayuda"


┌─────────────────────────────────────────────────────────────────┐
│ SOLUCIÓN DE PROBLEMAS COMUNES                                  │
└─────────────────────────────────────────────────────────────────┘

❌ PROBLEMA: "API key is invalid"
✅ SOLUCIÓN:
   1. Verifica que copiaste la key completa
   2. Debe empezar con "sk-"
   3. Revisa environment.ts
   4. Reinicia el servidor (npm start)

❌ PROBLEMA: El chatbot no aparece
✅ SOLUCIÓN:
   1. Verifica que estés logueado
   2. Refresca la página (F5)
   3. Revisa la consola (F12) para errores

❌ PROBLEMA: El micrófono no funciona
✅ SOLUCIÓN:
   1. Usa Chrome o Edge
   2. Permite el acceso al micrófono
   3. Verifica que tu micrófono funcione
   4. Habla claramente y en español

❌ PROBLEMA: "Cannot find module"
✅ SOLUCIÓN:
   1. cd diagram
   2. npm install
   3. npm start

❌ PROBLEMA: Puerto 4200 en uso
✅ SOLUCIÓN:
   1. Cierra otros procesos de Angular
   2. O usa: ng serve --port 4201
